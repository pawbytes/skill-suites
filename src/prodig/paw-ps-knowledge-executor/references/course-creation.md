# Course Creation

Build production-ready course materials with clear transformation, sellable structure, and tangible learner outcomes.

## Purpose

Transform product decisions and audience intelligence into a complete course architecture with curriculum, lesson outlines, supporting assets, and clear transformation arc that customers will pay for.

## Required Inputs

Before course creation, ensure these are available:

| Input | Source | Purpose |
|-------|--------|---------|
| Product Context | `curated/product-context.md` | What we're building and why |
| Audience Intelligence | `curated/audience-intelligence.md` | Who we're serving and their needs |
| Output Standards | `curated/output-standards.md` | Formatting and quality requirements |
| Knowledge Products Guidance | `curated/product-types/knowledge-products.md` | Product-type specific guidance |

If inputs are missing, escalate to appropriate agent before proceeding.

## Course Architecture Framework

### 1. Transformation Definition

Every course must answer: What transformation does the learner achieve?

**Transformation Statement Template:**

```
By the end of this course, {target learner} will be able to {specific outcome},
even if they currently {current state/objection}.
```

**Elements:**
- **Before state**: Where learners start
- **After state**: What learners can do
- **Bridge**: How the course gets them there
- **Objection handling**: Address common hesitations

### 2. Course Structure Options

Select based on transformation type and audience needs:

| Structure | Best For | Typical Length |
|-----------|----------|----------------|
| Linear Progression | Skill-building, step-by-step learning | 4-8 modules |
| Modular Toolkit | Reference-style, problem-solving | 6-12 modules |
| Cohort Journey | Community-based, time-bound | 4-6 modules + live |
| Project-Based | Portfolio building, tangible output | 3-5 modules + project |
| Certification Prep | Exam preparation, credential | 8-12 modules + practice |

### 3. Module Architecture

Each module follows a consistent structure:

```
Module {N}: {Module Title}

Module Outcome: {What learners will be able to do after this module}

Pre-lesson Check: {Self-assessment or readiness check}

Lessons:
- Lesson {N}.1: {Title} (5-15 min)
  - Learning objective
  - Core content
  - Key takeaway
  - Action item
- Lesson {N}.2: {Title}
- ...

Module Summary: Key concepts recap

Application Activity: Hands-on exercise applying module concepts

Module Assessment: Quiz, reflection, or checkpoint
```

### 4. Lesson Design

Each lesson follows micro-learning principles:

| Component | Purpose | Length |
|-----------|---------|--------|
| Hook | Why this matters to learner | 30-60 sec |
| Core Content | Main teaching | 5-12 min |
| Key Takeaway | One thing to remember | 30 sec |
| Action Item | Immediate application | 2-5 min |

### 5. Supporting Assets

Identify and plan for supporting materials:

| Asset Type | Purpose | When Required |
|------------|---------|---------------|
| Worksheets | Guided practice, exercises | Application activities |
| Templates | Ready-to-use frameworks | Implementation lessons |
| Checklists | Process following, QC | Step-by-step procedures |
| Cheat Sheets | Quick reference | Key concepts, formulas |
| Swipe Files | Examples, inspiration | Creative/strategic work |
| Resource Lists | Further learning | Deep-dive topics |
| Quizzes | Knowledge check | Each module |
| Capstone Project | Integration demonstration | Course completion |

## Production Process

### Phase 1: Architecture (30%)

1. Define transformation statement
2. Select course structure
3. Create module outline with outcomes
4. Map lesson count per module
5. Identify supporting assets
6. **Deliverable**: Course Architecture Document

### Phase 2: Lesson Planning (40%)

1. Write lesson titles and objectives for each module
2. Create lesson-by-lesson breakdown
3. Draft key takeaways and action items
4. Plan application activities
5. Design assessments
6. **Deliverable**: Detailed Lesson Outline

### Phase 3: Asset Specification (20%)

1. List all required assets by type
2. Specify purpose and use case for each
3. Provide briefs/templates for asset creation
4. Prioritize by importance
5. **Deliverable**: Asset Specification Document

### Phase 4: Production Notes (10%)

1. Delivery format recommendations (video, text, audio)
2. Platform considerations
3. Production checklist
4. Launch readiness criteria
5. **Deliverable**: Production Notes

## Quality Criteria

Before finalizing course materials, verify:

- [ ] Transformation statement is specific and compelling
- [ ] Module progression is logical and builds skills
- [ ] Each lesson has clear objective and takeaway
- [ ] Application activities provide hands-on practice
- [ ] Asset list is complete and prioritized
- [ ] Total time investment is appropriate for audience
- [ ] Course is sellable as structured (not "needs more later")
- [ ] A customer could start learning immediately after purchase

## Common Mistakes to Avoid

| Mistake | Fix |
|---------|-----|
| Too broad scope | Focus on specific transformation |
| No clear outcome | Every module needs an outcome |
| Information overload | Less is more; prioritize essentials |
| Missing application | Every concept needs practice |
| Weak transformation | Before/after must be meaningful |
| No differentiation | What makes this course unique? |
