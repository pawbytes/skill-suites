# Ebook/Guide Creation

Build production-ready written products with clear structure, reader journey, and sellable packaging.

## Purpose

Transform product decisions and audience intelligence into a complete written product with chapter structure, draft sections, and packaging notes that customers will pay for.

## Required Inputs

Before ebook/guide creation, ensure these are available:

| Input | Source | Purpose |
|-------|--------|---------|
| Product Context | `curated/product-context.md` | What we're building and why |
| Audience Intelligence | `curated/audience-intelligence.md` | Who we're serving and their needs |
| Output Standards | `curated/output-standards.md` | Formatting and quality requirements |
| Knowledge Products Guidance | `curated/product-types/knowledge-products.md` | Product-type specific guidance |

If inputs are missing, escalate to appropriate agent before proceeding.

## Ebook vs Guide Distinction

| Characteristic | Ebook | Guide |
|----------------|-------|-------|
| Purpose | Teach a concept or tell a story | Solve a specific problem |
| Structure | Narrative or thematic chapters | Process or framework-based |
| Length | 15,000-50,000 words | 5,000-15,000 words |
| Reading Experience | Cover to cover or reference | Quick implementation |
| Examples | "The Complete Guide to X", "Y Method" | "How to X in 30 Days", "Y Checklist" |

## Written Product Architecture

### 1. Reader Outcome Definition

Every written product must answer: What does the reader achieve?

**Outcome Statement Template:**

```
After reading this {ebook/guide}, {reader} will be able to {specific outcome},
without needing to {common barrier or alternative}.
```

### 2. Chapter/Section Structure

Select based on product type and reader needs:

| Structure | Best For | Organization |
|-----------|----------|--------------|
| Progressive | Skill-building, learning journey | Builds chapter by chapter |
| Reference | Problem-solving, lookup | Standalone sections |
| Framework | Methodology, system | Core framework + applications |
| Case Study | Proof-driven, example-based | Stories + analysis |
| Q&A/FAQ | Quick answers, troubleshooting | Question-based sections |

### 3. Chapter Architecture

Each chapter follows a consistent structure:

```
Chapter {N}: {Chapter Title}

Opening Hook: Why this chapter matters (1-2 paragraphs)

Core Content:
- Section 1: {Key concept}
- Section 2: {Key concept}
- Section 3: {Key concept}

Key Takeaways: 3-5 bullet points

Application: Exercise, reflection, or action item

Transition: Connection to next chapter
```

### 4. Content Elements

Plan for content variety within chapters:

| Element | Purpose | When to Use |
|---------|---------|-------------|
| Examples | Illustrate concepts | Every key point |
| Stories | Create connection, show proof | Opening hooks, transitions |
| Frameworks | Organize thinking | Core concepts |
| Checklists | Enable action | Processes, procedures |
| Templates | Accelerate implementation | Repeated tasks |
| Sidebars | Depth without derailing | Deep-dives, tangents |
| Quotes | Credibility, emphasis | Key principles |
| Stats/Data | Evidence, credibility | Claims, arguments |

### 5. Front and Back Matter

Plan the complete package:

**Front Matter:**
- Title page
- Copyright/disclaimer
- Dedication (optional)
- Table of contents
- Preface/Introduction

**Back Matter:**
- Conclusion/Summary
- About the author/brand
- Resources/Further reading
- Call to action
- Other products/services

## Production Process

### Phase 1: Outline (20%)

1. Define reader outcome statement
2. Select structure type
3. Create chapter list with titles
4. Define chapter outcomes
5. Identify key concepts per chapter
6. **Deliverable**: Chapter Outline Document

### Phase 2: Section Planning (30%)

1. Break each chapter into sections
2. Define section purposes
3. Identify content elements per section
4. Plan examples and stories
5. Draft key takeaways
6. **Deliverable**: Detailed Section Plan

### Phase 3: Draft Content (35%)

1. Write introduction/preface
2. Draft chapter content (outline level or full draft)
3. Create supporting elements (frameworks, lists)
4. Write conclusion
5. Plan front/back matter
6. **Deliverable**: Draft Manuscript

### Phase 4: Packaging Notes (15%)

1. Format recommendations (PDF, EPUB, print)
2. Design requirements
3. Cover brief
4. Launch checklist
5. **Deliverable**: Packaging Notes

## Quality Criteria

Before finalizing written product, verify:

- [ ] Reader outcome is specific and achievable
- [ ] Chapter progression is logical
- [ ] Each chapter has clear purpose
- [ ] Content variety maintains engagement
- [ ] Key takeaways are actionable
- [ ] Examples and frameworks support learning
- [ ] Introduction hooks the reader
- [ ] Conclusion provides clear next step
- [ ] Product is sellable as structured
- [ ] A customer could implement immediately after reading

## Writing Style Guidelines

| Principle | Application |
|-----------|-------------|
| Clarity over cleverness | Simple words, short sentences |
| Active voice | "You will learn" not "You will be taught" |
| Concrete examples | Specific, not abstract |
| Scannable | Headers, bullets, lists |
| Actionable | Every chapter enables doing |

## Common Mistakes to Avoid

| Mistake | Fix |
|---------|-----|
| Too much theory | Add examples and applications |
| No clear outcome | Define before writing |
| Inconsistent structure | Use chapter template |
| Missing hooks | Start chapters with "why this matters" |
| No action items | Add exercises, checklists, templates |
| Weak title | Test titles for clarity and appeal |
