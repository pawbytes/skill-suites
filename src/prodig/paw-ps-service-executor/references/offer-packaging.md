# Offer Packaging

## Purpose

Transform vague service ideas into structured, sellable offers with clear scope, outcomes, and boundaries.

## The Packaging Framework

### 1. Service Identity

**Name the service**
- Use benefit-driven or outcome-focused naming
- Avoid generic terms ("Consulting", "Coaching")
- Make it feel like a product

| Generic | Packaged |
|---------|----------|
| Marketing Consulting | 90-Day Launch Accelerator |
| Business Coaching | Scale-to-7 Framework |
| SEO Services | First Page Domination Program |
| Writing Services | Content Authority System |

**Tagline**
- One sentence that captures the transformation
- Format: "From [current state] to [desired state]"
- Example: "From invisible to industry authority in 90 days"

### 2. Core Promise

**The transformation statement**
What does the client walk away with? Be specific.

```
Service: [Name]

Promise: You will achieve [specific outcome] within [timeframe]

By the end of this service, you will have:
- [Tangible deliverable 1]
- [Tangible deliverable 2]
- [Tangible deliverable 3]
- [Intangible outcome 1]
- [Intangible outcome 2]
```

### 3. Target Client Profile

**Who is this for?**
Define the ideal client clearly. This helps selling AND delivery.

```
This service is designed for:
- [Role/identity]: e.g., "SaaS founders with 10-50 employees"
- [Pain point]: e.g., "Struggling to convert trial users to paid"
- [Stage]: e.g., "Post-product-market-fit, pre-scale"
- [Budget]: e.g., "Able to invest $5K-15K in growth"

This service is NOT for:
- [Who it doesn't serve]
- [Why it won't work for them]
```

### 4. Scope Boundaries

**What's included (In-Scope)**

List every element the client receives:

```
INCLUDED:
1. [Phase 1 deliverables]
   - [Specific item]
   - [Specific item]
2. [Phase 2 deliverables]
   - [Specific item]
3. [Ongoing elements]
   - [Communication cadence]
   - [Support level]
   - [Revision rounds]
```

**What's NOT included (Out-of-Scope)**

This is as important as inclusions. It prevents scope creep and sets expectations.

```
NOT INCLUDED:
- [Adjacent service that's separate]
- [Ongoing work beyond project scope]
- [Services requiring different expertise]
- [Typical add-ons]

Available as add-ons:
- [Add-on 1]: $[price]
- [Add-on 2]: $[price]
```

### 5. Service Components

**The Service Stack**

Every service has core components that define its structure:

| Component | Description | Example |
|-----------|-------------|---------|
| Core Delivery | Main work performed | Strategy + implementation |
| Client Touchpoints | Meetings, calls, reviews | Weekly calls, milestone reviews |
| Deliverables | Tangible outputs | Documents, templates, assets |
| Support | Between-session help | Slack access, email response time |
| Bonuses | Value-adds that sweeten the offer | Templates, training, access |

### 6. Delivery Phases

**Structure the journey**

Services work best when broken into phases:

```
PHASE 1: [Name] (Week 1-2)
Purpose: [What this phase accomplishes]
Activities:
- [Activity 1]
- [Activity 2]
Deliverables:
- [Deliverable 1]
Client Checkpoint: [Review/approval needed]

PHASE 2: [Name] (Week 3-4)
Purpose: [What this phase accomplishes]
Activities:
- [Activity 1]
- [Activity 2]
Deliverables:
- [Deliverable 1]
Client Checkpoint: [Review/approval needed]

[Continue for all phases]
```

### 7. Timeline and Duration

**Set realistic expectations**

```
Total Duration: [X weeks/months]

Key Milestones:
- Week [X]: [Milestone]
- Week [X]: [Milestone]
- Week [X]: [Milestone]

Client Time Investment:
- [Hours/week for calls]
- [Hours/week for homework/feedback]
- [Total client hours over project]
```

## Packaging Checklist

Before finalizing an offer:

- [ ] Has a marketable name
- [ ] Clear promise statement
- [ ] Target client defined
- [ ] Scope explicitly listed (in AND out)
- [ ] Phases structured with deliverables
- [ ] Timeline is realistic
- [ ] Can be delivered consistently by team (not just creator)
- [ ] Scope boundaries prevent common creep scenarios
- [ ] Value is clear compared to price

## Common Packaging Mistakes

| Mistake | Problem | Fix |
|---------|---------|-----|
| Vague scope | Client expectations diverge from reality | List every deliverable explicitly |
| No boundaries | Scope creep erodes margins | Define what's NOT included |
| Generic naming | Hard to market and remember | Use benefit-driven names |
| Unrealistic timeline | Delivery stress and missed deadlines | Add 20% buffer to estimates |
| Creator-dependent | Can't scale beyond you | Document process for team execution |
| Missing client investment | Client doesn't prepare | State required client time upfront |

- [Deliverable 2]

- [Deliverable 1]
- [Deliverable 2]

- [Support element 1]

## What's Not Included
- [Out of scope item 1]
- [Out of scope item 2]

## Timeline
[X] weeks from kickoff to completion
