# Onboarding Design

## Purpose

Design the client onboarding experience that sets expectations, gathers necessary information, and creates momentum for successful delivery.

## Why Onboarding Matters

| Poor Onboarding | Strong Onboarding |
|-----------------|-------------------|
| Confused clients | Clear expectations |
| Missing information | Complete inputs |
| Delayed start | Quick momentum |
| Scope misunderstandings | Aligned scope |
| Low client confidence | Trust established |

## Onboarding Components

### 1. Welcome Sequence

**Immediate (within 24 hours of signing):**

```
1. Welcome email
   - Thank you + excitement
   - What happens next
   - Key dates
   - Primary contact

2. Contract + invoice
   - Signed agreement
   - Payment confirmation
   - Receipt

3. Introduce team
   - Who they'll work with
   - Roles and responsibilities
   - How to reach each person
```

### 2. Information Gathering

**Client Input Request:**

```markdown
## Getting Started

To hit the ground running, please complete the following:

### Due: [Date - typically 3-5 days before kickoff]

**1. Questionnaire** ([Link])
- Business context
- Goals and priorities
- Stakeholder information
- Timeline constraints
- Success criteria

**2. Access & Assets**
- [ ] [Platform 1] access (instructions: [link])
- [ ] [Platform 2] access (instructions: [link])
- [ ] Brand guidelines (if applicable)
- [ ] Existing materials (links/folders)

**3. Stakeholders**
- Key decision-makers
- Their roles in this project
- Preferred communication style

**4. Schedule Preferences**
- Preferred meeting times
- Time zone
- Any travel/blackout dates
```

### 3. Kickoff Meeting

**Standard Agenda (60 minutes):**

```markdown
## Kickoff Meeting Agenda

**Attendees:** [List]

**Duration:** 60 minutes

**Agenda:**

1. Introductions (5 min)
   - Team members
   - Roles and communication

2. Project Overview (10 min)
   - Goals and success criteria
   - Scope confirmation
   - Timeline review

3. Process Walkthrough (15 min)
   - How we work
   - Communication channels
   - Review/approval process
   - Revision handling

4. Information Gathering (20 min)
   - Questions from questionnaire
   - Context deep-dive
   - Priorities discussion

5. Next Steps (10 min)
   - Immediate actions
   - First milestone
   - Next meeting

**Outputs:**
- Meeting notes
- Confirmed timeline
- Action items
```

### 4. Project Setup

**Internal Setup Checklist:**

```
□ Create project folder structure
□ Set up project management tool
□ Create client communication channel
□ Schedule milestone meetings
□ Prepare first phase deliverables
□ Brief team on client context
□ Set up time tracking
```

**Client-Facing Setup:**

```
□ Shared folder access
□ Project dashboard/view access
□ Calendar invites sent
□ Welcome packet delivered
□ Contact information shared
```

## Onboarding Automation

### Automatable Elements

| Element | Automation Level | Tool |
|---------|------------------|------|
| Welcome email | Full | Email automation |
| Questionnaire delivery | Full | Typeform, Google Forms |
| Calendar scheduling | Full | Calendly, Cal.com |
| Folder creation | Partial | Templates + Zapier |
| Invoice/payment | Full | Stripe, FreshBooks |
| Project setup | Partial | Templates |

### Automation Sequence

```
Day 0: Contract signed
  → Welcome email sent automatically
  → Questionnaire link sent
  → Kickoff calendar invite sent

Day 1-3: Client completes questionnaire
  → Responses captured in project tool
  → Team notified

Day 3-5: Pre-kickoff prep
  → Team reviews responses
  → Prepares kickoff deck

Day 5-7: Kickoff meeting
  → Meeting held
  → Notes captured and shared
  → Project officially begins
```
