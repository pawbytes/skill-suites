# Delivery Design

## Purpose

Define how a service is fulfilled — the operational model that ensures consistent, quality delivery every time.

## Delivery Models

### 1. Milestone-Based (Project)

**Best for:** Fixed-scope projects with clear beginning and end

```
Structure:
Kickoff → Milestone 1 → Milestone 2 → Milestone N → Completion

Characteristics:
- Fixed scope defined upfront
- Payment tied to milestones
- Clear beginning and end
- Success = project delivered
```

**Example:**
```
"Website Redesign Package"
Kickoff (Week 1) → Design Review (Week 3) → Build Complete (Week 6) → Launch (Week 8)
Payments: 40% upfront, 30% at design approval, 30% at launch
```

### 2. Retainer (Ongoing)

**Best for:** Ongoing access, recurring work, or advisory relationships

```
Structure:
Monthly recurring → Defined deliverables each period → Rollover policy

Characteristics:
- Predictable revenue
- Ongoing relationship
- Defined monthly scope
- Success = sustained value delivery
```

**Example:**
```
"Marketing Partner Retainer"
Monthly: $5,000
Includes:
- 2 strategy calls/month
- Up to 20 hours execution
- Priority Slack access
- Monthly reporting
Unused hours: Do not roll over (or roll over up to 10 hrs for 1 month)
```

### 3. Sprint (Intensive)

**Best for:** Focused, short-duration work with concentrated effort

```
Structure:
Pre-work → Sprint period → Delivery → Debrief

Characteristics:
- Intense, focused timeframe
- Requires full client availability
- High value density
- Success = major progress in short time
```

**Example:**
```
"2-Day Strategy Sprint"
Pre-work: Audit questionnaire (client completes 1 week before)
Day 1: Deep dive, analysis, strategy development
Day 2: Roadmap creation, prioritization, documentation
Delivery: Strategy deck + 90-day roadmap
Debrief: 30-min follow-up call (1 week later)
```

### 4. Cohort-Based

**Best for:** Services delivered to groups with shared timeline

```
Structure:
Enrollment period → Cohort start → Weekly sessions → Completion

Characteristics:
- Leverage (one-to-many delivery)
- Built-in community
- Fixed schedule
- Success = cohort outcomes
```

**Example:**
```
"Agency Accelerator"
Enrollment: Open for 2 weeks
Cohort: 8-week program
Sessions: Weekly 90-min group calls + Q&A
Between sessions: Assignments + peer feedback
Completion: Certification + showcase
```

### 5. Tiered (Good/Better/Best)

**Best for:** Serving different client segments with one service concept

```
Structure:
Tier 1 (DIY) → Tier 2 (Done-With-You) → Tier 3 (Done-For-You)

Characteristics:
- Multiple entry points
- Clear upgrade path
- Same core promise, different delivery depth
- Success = right fit for each tier
```

**Example:**
```
"Funnel Builder"

DIY ($997):
- Complete funnel template pack
- Video training library
- Email support

Done-With-You ($2,997):
- Everything in DIY
- 3 strategy calls
- Funnel review + feedback
- 2 revision rounds

Done-For-You ($9,997+):
- Everything in DWY
- Complete build by our team
- Copy + design + tech setup
- 30 days of optimization
```

## Delivery Design Elements

### Client Touchpoints

Define every point of client interaction:

| Touchpoint | Frequency | Duration | Purpose |
|------------|-----------|----------|---------|
| Kickoff call | Once | 60 min | Align on goals, scope, process |
| Weekly sync | Weekly | 30 min | Progress update, questions |
| Milestone review | Per milestone | 45 min | Approve deliverables |
| Async updates | As needed | - | Status, questions via email/Slack |
| Final debrief | Once | 30 min | Review outcomes, next steps |

## Delivery Process Documentation

### Process Flow

```
1. SALE → Contract signed, payment received
2. ONBOARDING → Welcome packet, kickoff scheduled
3. KICKOFF → Goals confirmed, timeline set
4. EXECUTION → [Phase-specific work]
5. REVIEW → Client feedback on deliverables
6. REVISION → [Number] revision rounds
7. DELIVERY → Final deliverables handed off
8. DEBRIEF → Outcomes reviewed, next steps discussed
9. CLOSE → Project archived, testimonial request
```

### Revision Policy

```
Included: [X] rounds of revisions per deliverable
Revision process:
1. Client provides consolidated feedback
2. Team implements changes
3. Client approves or provides next round

Out of scope (additional cost):
- Changes to original scope
- Reopening approved deliverables
- Delays beyond [X] days for client feedback
```

### Quality Checkpoints

Define quality gates before client delivery:

```
Before any deliverable leaves the team:
□ Peer review completed
□ Quality checklist verified
□ Formatting consistent
□ Links/tested
□ Client context considered
□ Delivery email drafted
```

## Delivery Automation

### Automatable Elements

| Element | Automation Level | Tool Examples |
|---------|------------------|---------------|
| Scheduling | Full | Calendly, Cal.com |
| Onboarding | High | Typeform + Zapier |
| Status updates | Medium | Project tool automation |
| Invoicing | Full | Stripe, FreshBooks |
| File delivery | High | Drive, Dropbox |
| Check-ins | Low | Still needs human touch |

### Automation Principles

1. Automate logistics, not relationships
2. Templates reduce decision fatigue
3. Systems enable scale, not replace quality
4. Document before automating
5. Human touchpoints at high-value moments
