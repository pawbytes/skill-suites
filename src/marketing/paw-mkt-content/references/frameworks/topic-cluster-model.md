# Topic Cluster Model

### What It Is
A strategic approach to organizing site content around broad "pillar" topics, supported by tightly related "cluster" articles that interlink to build topical authority.

### Structure
```
                    [Pillar Page]
                   /   |    |    \
                  /    |    |     \
          [Cluster] [Cluster] [Cluster] [Cluster]
            |   \      |         |        /   |
          [Sub] [Sub] [Sub]    [Sub]   [Sub] [Sub]
```

### Pillar Page Requirements
- **Length**: 3,000-5,000+ words
- **Scope**: Comprehensive coverage of a broad topic
- **Format**: Ultimate guide, definitive resource, or complete playbook
- **Links out**: To every cluster page in the topic group
- **CTA**: Lead magnet or conversion action relevant to the whole topic
- **Update cadence**: Quarterly review, semi-annual refresh

### Cluster Content Requirements
- **Length**: 1,200-2,500 words
- **Scope**: One specific subtopic or question within the pillar
- **Links back**: Must link to the pillar page (contextual, not forced)
- **Cross-links**: Link to 2-3 sibling cluster pages where relevant
- **Intent match**: Each cluster targets a distinct keyword/search intent

### How to Map a Topic Cluster

**Step 1: Choose the pillar topic**
- Must align with a core product/service offering
- Must have sufficient search volume (1,000+ monthly searches)
- Must be broad enough to support 8-15 subtopics

**Step 2: Identify cluster subtopics**
- Use keyword research (Ahrefs, SEMrush, Google Keyword Planner)
- Mine "People Also Ask" and "Related Searches" in Google
- Analyze competitor content structure
- Survey customers for their questions

**Step 3: Map search intent per cluster**
- Informational: "What is X", "How does X work"
- Navigational: "X tool", "X platform"
- Commercial: "Best X for Y", "X vs Z"
- Transactional: "Buy X", "X pricing", "X free trial"

**Step 4: Build the internal linking architecture**
- Every cluster links to the pillar (mandatory)
- Pillar links to every cluster (mandatory)
- Clusters cross-link where contextually relevant (recommended)
- Use descriptive anchor text, not "click here"

### Example: SaaS Project Management Tool
```
Pillar: "The Complete Guide to Project Management" (4,000 words)
  |
  |-- Cluster: "Agile vs Waterfall: Which Method Is Right for Your Team?" (1,800 words)
  |-- Cluster: "How to Create a Project Timeline in 7 Steps" (1,500 words)
  |-- Cluster: "10 Best Project Management Templates for 2026" (2,200 words)
  |-- Cluster: "Project Risk Management: A Practical Framework" (1,600 words)
  |-- Cluster: "How to Run an Effective Sprint Retrospective" (1,400 words)
  |-- Cluster: "Resource Allocation: Balancing Workloads Across Teams" (1,700 words)
  |-- Cluster: "Project Management for Remote Teams: Complete Playbook" (2,000 words)
  |-- Cluster: "Gantt Charts Explained: When and How to Use Them" (1,300 words)
  |-- Cluster: "Project Management Software Comparison 2026" (2,500 words)
  |-- Cluster: "How to Write a Project Brief That Keeps Teams Aligned" (1,200 words)
```

### Measuring Cluster Performance
- Track pillar page rankings for head terms (monthly)
- Track cluster rankings for long-tail terms (monthly)
- Monitor internal link click-through using GA4 events
- Measure topical authority improvement via total organic impressions for the cluster
