# Demo Scripts Capability

**Outcome:** Build customized demo talk tracks that show 3-4 workflows tied to prospect's stated problems, not generic feature tours.

---

## When to Use

Preparing for specific demos, coaching new reps, building demo scripts for different personas or use cases.

---

## Pre-Demo Prep

Before screen share, confirm:

- Prospect's specific goals from discovery
- Who's in the room (economic buyer, technical buyer, champion, end user)
- Demo environment with their industry/use case/actual data if possible
- Agenda: "5 min recap, 30 min showing, 10 min Q&A"

---

## Demo Structure

| Phase | Duration | Purpose |
|---|---|---|
| Discovery Recap | 2-3 min | Confirm goals and pains before showing anything |
| Workflow Walkthroughs | 15-25 min | Show 3-4 workflows tied to named problems |
| Value Summary | 3-5 min | Connect features to their problems with ROI |
| Discovery and Close | 5-10 min | Ask what they need to move forward |

---

## Phase-by-Phase Principles

**Discovery Recap:**
- Restate their goal, problem, impact
- Wait for confirmation before proceeding

**Workflow Walkthroughs:**
- For each: state problem → show feature → quantify impact → ask "How does this compare to today?"
- Pause after each major feature

**Value Summary:**
- Tie each feature to specific pain
- Close with aggregate ROI estimate

**Discovery and Close:**
- Ask open questions: "What would need to be true for you to move forward?"
- Resist filling silence

---

## Demo Best Practices

- Show, don't tell — if you can show it, don't describe it
- Check in after every major feature
- If tech fails: "Let me pull that up differently" — move on, don't apologize extensively
- Use their terminology — if they say "deals", say "deals"
- Aim for 50/50 talk time — ask questions
- Missing feature: "That's on roadmap for [timeframe]. Here's how customers handle it today."
