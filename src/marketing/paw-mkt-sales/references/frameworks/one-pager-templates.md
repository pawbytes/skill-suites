# One-Pager Templates

Fill-in templates for product one-pagers, executive summary one-pagers, and technical briefs. Each is designed to be scannable in 30 seconds and usable as a champion selling tool.

---

### 2.1 Product One-Pager

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[PRODUCT NAME] — [ONE-LINE VALUE PROPOSITION, MAX 12 WORDS]
                                              [BRAND LOGO]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

THE PROBLEM
[2-3 sentences. The specific pain in customer language. Quantify if possible.]

THE SOLUTION
• [Outcome-focused point 1. "Automatically [action] so your team can [result]."]
• [Outcome-focused point 2.]
• [Outcome-focused point 3.]

WHY [PRODUCT NAME]
• [Differentiator 1] → [Measurable result or proof]
• [Differentiator 2] → [Measurable result or proof]
• [Differentiator 3] → [Measurable result or proof]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
"[STRONG CUSTOMER QUOTE — specific outcome, attributed to name/role/company]"

[CUSTOMER LOGO 1]  [CUSTOMER LOGO 2]  [CUSTOMER LOGO 3]  [CUSTOMER LOGO 4]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[CTA — single action]: [URL or contact method]
[BRAND WEBSITE] | [BRAND EMAIL] | [TRIAL/DEMO LINK]
```

**Formatting rules:** Single page. Two-column layout if content permits. No paragraphs in body — bullet points only. Minimum 10pt font for print.

### 2.2 Executive Summary One-Pager

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[OUTCOME HEADLINE] for [COMPANY NAME]
                                              [BRAND LOGO]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

THE BUSINESS PROBLEM
[2 sentences. What challenge their team is solving. Quantify the cost if possible.
 Ideally use language their champion used in conversation.]

PROPOSED SOLUTION
[1 sentence. What [PRODUCT NAME] does. No jargon. Written from their perspective.]

EXPECTED OUTCOMES
Based on results with similar [COMPANY TYPE / INDUSTRY] organizations:
• [OUTCOME 1]: [metric], [timeframe] — [customer reference if permitted]
• [OUTCOME 2]: [metric], [timeframe]
• [OUTCOME 3]: [metric], [timeframe]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

INVESTMENT AND RETURN
  Annual investment:     $[PRICE]
  Expected year-1 value: $[ROI AMOUNT] (based on [KEY ASSUMPTION])
  Payback period:        [X] months
  3-year cumulative:     $[3-YEAR VALUE] net of investment

IMPLEMENTATION
  Time to live:          [X weeks]
  Team hours required:   [X hours from their side]
  Key dependencies:      [DATA MIGRATION / SSO / API — or "None"]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

RECOMMENDED NEXT STEP
[CHAMPION NAME] and [YOUR NAME] recommend [SPECIFIC ACTION] by [DATE].

Questions? [YOUR NAME] | [EMAIL] | [PHONE/CALENDAR LINK]
```

### 2.3 Technical Brief Template

```
# [PRODUCT NAME] — Technical Overview

**Prepared for:** [COMPANY NAME] — [TECHNICAL CONTACT NAME, TITLE]
**Date:** [DATE]
**Prepared by:** [YOUR NAME, TITLE]

---

## Architecture Overview
[1 paragraph. Deployment model: SaaS/cloud/on-prem/hybrid. Infrastructure.
 Uptime SLA. Redundancy.]

## Security and Compliance
| Requirement | Status |
|---|---|
| SOC 2 Type II | [Certified / In progress / N/A] |
| GDPR | [Compliant / Details available] |
| ISO 27001 | [Certified / N/A] |
| HIPAA | [Compliant / N/A — check with customer] |
| SSO / SAML | [Supported / Roadmap / N/A] |
| Data residency | [US / EU / Customer-specified] |
| Penetration testing | [Annual / Semi-annual — last date] |

Security documentation available on request: [LINK TO SECURITY PAGE OR CONTACT]

## Integrations
**Native integrations:**
- [INTEGRATION 1] — [What data flows, bidirectional or one-way]
- [INTEGRATION 2]

**API:**
[REST / GraphQL]. [Authentication method]. [Rate limits]. [Docs link.]

**Webhooks:** [Yes/No — what events trigger]

## Data Handling
- Data encryption: [In transit: TLS 1.2+] [At rest: AES-256]
- Data export: [Formats available] [How triggered]
- Data deletion: [Retention period] [Deletion process on cancellation]
- Subprocessors: [List or link to subprocessor page]

## Implementation Requirements
| Item | Detail |
|---|---|
| Implementation time | [X weeks] |
| Required from customer | [Hours, roles, systems access] |
| Data migration | [Supported / Self-serve / Professional services] |
| Training | [Included / Available as add-on] |

## Uptime and Support
- SLA: [X% uptime commitment]
- Status page: [URL]
- Support tiers: [Community / Email / Priority / Dedicated CSM]
- Support hours: [Coverage hours and timezone]
- Incident response: [P1 response time commitment]

## References
Available upon request for organizations of similar size and technical complexity.
Contact: [CUSTOMER SUCCESS EMAIL]
```
