# Content Library Capability

**Outcome:** Organize sales collateral so reps can find any asset in under 2 minutes, with clear version control and tagging.

---

## When to Use

Setting up new sales library, reorganizing existing assets, creating asset index, retiring stale content.

---

## Folder Structure

```
{resolved-path}/
  decks/
  one-pagers/
  objection-handling/
  demo-scripts/
  roi-calculator/
  champion-kits/
  research/
```

---

## Asset Tagging

Every asset tagged for quick filtering:

- **Stage:** Awareness / Discovery / Evaluation / Proposal / Closing / Post-close
- **Persona:** Economic buyer / Technical buyer / Champion / End user
- **Format:** Deck / One-pager / Calculator / Script / Battle card / Email
- **Vertical:** [Industry names relevant to brand]
- **Status:** Current / Under review / Archived

---

## Library Index

Create `SALES-LIBRARY-INDEX.md` at top level with table:

| Asset Name | Stage | Persona | Format | Status | Path | Last Updated |
|---|---|---|---|---|---|---|

---

## Version Control

- Include date in filename: `{asset-name}-{YYYY-MM-DD}`
- Archive superseded versions to `/archive/` — never delete
- Update stats/proof within 48 hours when they change
- Notify sales team on major updates — don't rely on them checking

---

## Template

`./frameworks/sales-library-index.md`

---

## Output

Deliver index to: `{resolved-path}/SALES-LIBRARY-INDEX.md`
