# PIE Framework (Goward)

**What it is:** A CRO-specific prioritisation framework scoring pages or experiments on Potential, Importance, and Ease.

**SOSTAC Phase:** Tactics

**Scoring (1-10 each):**
- **Potential:** How much improvement is possible? (low-converting pages score higher)
- **Importance:** How much traffic / revenue does this page influence?
- **Ease:** How technically and politically easy is it to run the test?

`PIE Score = (Potential + Importance + Ease) / 3`

**PIE vs ICE:** Use PIE when prioritising CRO experiments on specific pages. Use ICE for a broader mixed-tactic backlog that includes content, paid, email, and product changes.
