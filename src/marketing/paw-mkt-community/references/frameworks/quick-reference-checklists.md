# Quick-Reference Checklists

## Pre-Launch Checklist

- [ ] Community purpose defined (1-2 sentences)
- [ ] Target member persona documented
- [ ] Platform selected and configured
- [ ] Channel/space structure set up (5-7 channels maximum)
- [ ] Roles and permissions configured
- [ ] Community guidelines published
- [ ] Onboarding flow designed and tested
- [ ] Moderation tools configured
- [ ] 10-15 seed posts written
- [ ] 3-5 resource threads published
- [ ] Welcome guide published
- [ ] Founding member list created (50-100 names)
- [ ] Founding member invitation messages drafted
- [ ] Launch event planned
- [ ] Success metrics and targets defined
- [ ] Analytics tracking set up

## Launch Week Checklist

- [ ] Founding members briefed and ready to welcome newcomers
- [ ] Moderators trained and scheduled
- [ ] Launch announcements sent (email, social, website, in-product)
- [ ] Launch event executed
- [ ] 48-hour welcome challenge active
- [ ] Every new member introduction gets a response within 4 hours
- [ ] Daily metrics tracked (joins, intros, posts, event attendance)
- [ ] Onboarding bottlenecks identified and fixed in real-time
- [ ] Community highlights published at end of launch week

## Weekly Operations Checklist

- [ ] Review health dashboard metrics
- [ ] Post 3-5 discussion prompts
- [ ] Host at least one community event
- [ ] Respond to all unanswered questions
- [ ] Welcome all new members
- [ ] Moderate flagged content
- [ ] Check for at-risk members (active but declining engagement)
- [ ] Update content calendar for next week
- [ ] Sync with moderator team
- [ ] Repurpose at least one piece of community content for external channels
