# Platform Specifications Reference

Technical specifications ensure designs render correctly on their intended platforms. Always design at exact specifications to avoid cropping, compression, or quality issues.

---

## Social Media Image Dimensions

### Instagram

| Format | Dimensions | Aspect Ratio | Notes |
|--------|------------|--------------|-------|
| **Feed Post (Square)** | 1080 × 1080 px | 1:1 | Standard format |
| **Feed Post (Portrait)** | 1080 × 1350 px | 4:5 | Occupies more screen space |
| **Feed Post (Landscape)** | 1080 × 608 px | 1.91:1 | Less vertical presence |
| **Carousel** | 1080 × 1080 px or 1080 × 1350 px | 1:1 or 4:5 | All slides must match aspect ratio |
| **Stories** | 1080 × 1920 px | 9:16 | Full vertical screen |
| **Reels Cover** | 1080 × 1920 px | 9:16 | Portrait cover image |
| **Profile Picture** | 320 × 320 px | 1:1 | Displayed at 110 × 110 px |
| **IGTV Cover** | 420 × 654 px | 1:1.55 | Channel cover |
| **Highlights Cover** | 1080 × 1920 px | 9:16 | Cropped to circle |

**Instagram Best Practices:**
- Portrait (4:5) posts get 25% more screen real estate than square
- Carousels support up to 20 slides (expanded from 10)
- Keep text at least 60px from edges to avoid cropping
- Use 24pt minimum font size for mobile readability
- Mixed media carousels (images + videos) get highest engagement

|-------------|------------|------------|-------------|

| **Large Format** | 150-200 DPI | CMYK | PDF, TIFF |

| **Billboard** | 72-150 DPI | CMYK | PDF, TIFF |

### Bleed, Trim, and Safety

```
┌─────────────────────────────────────┐
│              BLEED AREA              │  ← Extend background to here
│   (3mm / 0.125" beyond trim)        │     (gets cut off)
│  ┌───────────────────────────────┐  │
│  │         TRIM SIZE             │  │  ← Final cut size
│  │                               │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │     SAFETY MARGIN       │  │  │  ← Keep critical content
│  │  │     (3mm from trim)     │  │  │     inside this zone
│  │  │                         │  │  │
│  │  └─────────────────────────┘  │  │
│  │                               │  │
│  └───────────────────────────────┘  │
│                                     │
└─────────────────────────────────────┘
```

**Standard Margins:**
- **Bleed:** 3mm (0.125") beyond trim on all sides
- **Safety zone:** 3mm (0.125") inside trim on all sides
- **Gutter:** Additional margin for bound materials

### Print Color Guidelines

**CMYK Color Mode:**
- All print files must be CMYK
- Convert RGB to CMYK before export
- Expect slight color shift when converting

**Color Profiles:**
- **US:** U.S. Web Coated (SWOP) v2
- **Europe:** Coated FOGRA39
- **Newsprint:** US Newsprint (SNAP) 2006

**Black Values:**
- **Rich Black:** C60 M40 Y40 K100 (for large areas)
- **Standard Black:** K100 (for text and small elements)
- Never use registration black (C100 M100 Y100 K100)

### Print File Checklist

- [ ] Correct dimensions including bleed
- [ ] 300 DPI resolution at final size
- [ ] CMYK color mode
- [ ] All fonts outlined or embedded
- [ ] Images embedded (not linked)
- [ ] Bleed extends beyond trim
- [ ] Critical content inside safety zone
- [ ] Spell check completed
- [ ] Color profile assigned

## Digital/Web Specifications

### Web Image Standards

| Use Case | Format | Resolution | Optimization |
|----------|--------|------------|--------------|
| **Website hero** | WebP/JPG | 1920px+ width | Compress to <200KB |
| **Product image** | WebP/PNG | 800-1200px | Keep sharp, allow zoom |
| **Thumbnail** | JPG/WebP | 300-400px | High compression OK |
| **Icon** | SVG/PNG | Multiple sizes | Lossless |
| **Logo** | SVG | Vector | Infinite scalability |
| **Background** | WebP/JPG | 1920px+ | Heavy compression OK |

### Retina/High-DPI Considerations

For sharp display on retina screens:
- Export at 2x dimensions (e.g., 800px image for 400px display)
- Use SVG for icons and logos when possible
- Compress heavily; retina screens hide compression artifacts

### Web Safe Zones

| Platform | Safe Zone | Why |
|----------|-----------|-----|
| **Browser** | Top 40px hidden by UI | Browser chrome |
| **Mobile web** | Bottom 100px hidden | Browser controls |
| **App store screenshots** | Device-specific overlays | Store UI elements |

## Export Format Quick Reference

| Need | Best Format | Why |
|------|-------------|-----|
| **Transparency needed** | PNG | Alpha channel support |
| **Photography/Web** | JPG/WebP | Best compression for photos |
| **Print** | PDF/TIFF | Preserves quality, CMYK support |
| **Logo/Icon** | SVG | Scalable, small file size |
| **Animation** | GIF/MP4 | GIF for simple, MP4 for quality |
| **Web performance** | WebP | Better compression than JPG/PNG |
| **Social media** | JPG/PNG | Universal compatibility |
