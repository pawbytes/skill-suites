# Design Methodology Framework

The Designer follows a structured methodology adapted from industry-proven frameworks. This ensures every output meets professional production standards.

## The Design Process: Five-Phase Framework

### Phase 1: Discovery & Brief Analysis

**Objective:** Understand the problem before solving it.

Every design task begins by extracting requirements from the brief:

1. **Identify deliverable type** — Social post, carousel, flyer, brand asset, or multi-platform variant
2. **Extract specifications** — Platform, dimensions, format requirements
3. **Understand context** — Campaign goals, target audience, placement
4. **Review brand guidelines** — Colors, typography, voice, visual identity
5. **Define success criteria** — What does "done" look like?

**Key questions to answer:**
- What platform(s) will this appear on?
- What action should viewers take?
- What existing assets must be incorporated?
- What constraints exist (time, tools, brand rules)?

### Phase 2: Ideation & Concept Development

**Objective:** Generate multiple solutions before committing.

**Ideation Process:**

1. **Research references** — Search for design inspiration relevant to the brief
2. **Moodboarding** — Collect visual references for color, style, composition
3. **Thumbnail sketching** — Rapid exploration of layout options (3-5 minimum)
4. **Concept selection** — Choose the strongest concept based on brief alignment

**Divergent-to-Convergent Pattern:**
- Start broad: generate many ideas without judgment
- Then narrow: evaluate against brief requirements
- Select: one strong concept to develop

### Phase 3: Design Execution

**Objective:** Transform concept into production-ready asset.

**Execution Checklist:**

1. **Set up canvas correctly**
   - Exact dimensions for target platform
   - Correct color mode (RGB for digital, CMYK for print)
   - Proper resolution (72-150 PPI digital, 300 DPI print)
   - Bleed setup for print (0.125" / 3mm standard)

2. **Build visual hierarchy**
   - Primary element: What must be seen first (largest, highest contrast)
   - Secondary elements: Supporting information (medium visual weight)
   - Tertiary elements: Details and fine print (subtle treatment)

3. **Apply brand guidelines**
   - Exact color values (HEX for digital, CMYK/Pantone for print)
   - Approved typography (font families, weights, sizes)
   - Logo placement rules (clear space, minimum size)
   - Visual style consistency (photography style, illustration style)

4. **Ensure technical quality**
   - Images at correct resolution (no upscaling artifacts)
   - Text readable at intended viewing size
   - Safe zones respected (no critical content near edges)
   - Export format optimization (PNG for transparency, JPG for photos)

### Phase 4: Quality Review

**Objective:** Verify output meets professional standards.

**The Squint Test:**
Blur your vision and observe what stands out. The most important element should remain visible. If something secondary dominates, hierarchy needs adjustment.

**Brand Consistency Check:**
- [ ] Colors match brand palette exactly
- [ ] Typography follows brand guidelines
- [ ] Logo usage complies with brand rules
- [ ] Visual style aligns with brand identity

**Technical Quality Check:**
- [ ] Correct dimensions for platform
- [ ] No pixelation or compression artifacts
- [ ] Text legible at intended viewing size
- [ ] File format appropriate for use case
- [ ] File size optimized (no unnecessary bloat)

### Phase 5: Export & Delivery

**Objective:** Package assets for their intended use.

**Export Standards by Use Case:**

| Use Case | Format | Resolution | Color Mode |
|----------|--------|------------|------------|
| Social Media | PNG/JPG | 1080px+ width | RGB |
| Web/Digital | PNG/WebP | 2x for retina | RGB |
| Print (standard) | PDF/PNG | 300 DPI | CMYK |
| Print (large format) | PDF/TIFF | 150-300 DPI | CMYK |
| Video overlay | PNG | 1920x1080+ | RGB |
| App/UI | PNG/SVG | Multiple @x | RGB |

## Visual Hierarchy Principles

Visual hierarchy determines what viewers see first, second, and third. It transforms a collection of elements into a guided experience.

### Primary Tools for Creating Hierarchy

| Tool | Effect | Application |
|------|--------|-------------|
| **Size** | Largest elements attract most attention | Headlines dominant, body text subordinate |
| **Contrast** | Difference from surroundings draws eye | CTA buttons, highlighted text, focal images |
| **Position** | Top-left receives most attention (Western reading) | Key messages above the fold, important content first |
| **Color** | Saturation and hue differences create emphasis | Brand colors for headlines, neutral for supporting |
| **Spacing** | Isolation creates emphasis | White space around key messages |
| **Typography** | Weight, style, and scale differentiate levels | Bold headings, regular body, light captions |

### Hierarchy Levels

A clear hierarchy typically has 3-5 distinct levels:

1. **Primary** — The single most important element (headline, hero image, main CTA)
2. **Secondary** — Supporting headlines, key visuals, important information
3. **Tertiary** — Body text, descriptions, details
4. **Quaternary** — Captions, footnotes, metadata, fine print

**Rule:** Every element should clearly belong to one level. Ambiguity creates confusion.

### The 60-30-10 Color Rule

For balanced color distribution:
- **60% dominant color** — Background, primary surfaces
- **30% secondary color** — Supporting elements, cards, sections
- **10% accent color** — CTAs, highlights, emphasis

## Composition Fundamentals

### Rule of Thirds

Divide the canvas into a 3x3 grid. Place key elements at intersection points or along grid lines. This creates natural balance and visual interest.

**Application:**
- Position main subjects at intersection points
- Align horizons with horizontal grid lines
- Place vertical elements along vertical grid lines

### Golden Ratio (1:1.618)

A mathematical proportion found in nature that creates inherently pleasing compositions.

**Application:**
- Use for logo proportions and layout ratios
- Create spiral compositions for dynamic designs
- Determine content area vs. whitespace ratios

### Visual Weight Balance

Every element has visual weight based on:
- Size (larger = heavier)
- Color intensity (saturated = heavier)
- Complexity (detailed = heavier)
- Position (lower = heavier, right = heavier in Western culture)

**Balance types:**
- **Symmetrical** — Formal, stable, traditional
- **Asymmetrical** — Dynamic, modern, engaging
- **Radial** — Draws eye to center

### Gestalt Principles

These describe how viewers perceive visual groups:

| Principle | Meaning | Design Application |
|-----------|---------|-------------------|
| **Proximity** | Elements close together are perceived as related | Group related content, separate distinct sections |
| **Similarity** | Elements sharing visual traits are perceived as related | Consistent styling for same-level content |
| **Continuity** | Eye follows continuous lines and curves | Guide eye through content flow |
| **Closure** | Mind completes incomplete shapes | Minimalist logos, implied shapes |
| **Figure-Ground** | Elements are perceived as foreground or background | Create clear focal points, avoid confusion |

## Typography System

### Type Scale Construction

A type scale creates consistent size relationships. Use a modular ratio (1.25 or 1.333 work well):

| Level | Size (base 16px, ratio 1.333) | Usage |
|-------|-------------------------------|-------|
| Display | 64px | Hero headlines |
| H1 | 48px | Page titles |
| H2 | 36px | Section headings |
| H3 | 27px | Subheadings |
| Body | 16px | Paragraph text |
| Small | 12px | Captions, footnotes |

### Font Pairing Guidelines

**Contrast principle:** Paired fonts should have noticeable differences.

| Pairing Strategy | Example | Best For |
|-----------------|---------|----------|
| Serif + Sans-serif | Playfair Display + Open Sans | Elegant, professional |
| Bold + Light | Montserrat Bold + Montserrat Light | Modern, clean |
| Condensed + Standard | Bebas Neue + Roboto | Impact, headlines |
| Decorative + Neutral | Pacifico + Lato | Personality, casual |

**Limit:** Maximum 2-3 font families per design to avoid visual chaos.

### Line Length & Spacing

| Measurement | Optimal Range | Why It Matters |
|-------------|---------------|----------------|
| Line length | 45-75 characters | Too long = difficult to track; too short = choppy |
| Line height | 1.4-1.6x font size | Prevents lines from crowding |
| Paragraph spacing | 1-1.5x font size | Creates clear separation |

## Design Quality Standards

### Professional Quality Indicators

A production-grade design exhibits:

1. **Intentional hierarchy** — Clear visual priority, no ambiguity
2. **Brand alignment** — Consistent with established brand identity
3. **Technical precision** — Correct specs, clean export, no artifacts
4. **Appropriate complexity** — Not oversimplified, not overdesigned
5. **Clear communication** — Message understood without confusion
6. **Visual polish** — Refined details, no rough edges or mistakes

### Common Quality Failures to Avoid

| Failure | Problem | Fix |
|---------|---------|-----|
| **Everything bold** | No hierarchy, all elements compete | Choose one emphasis element |
| **Competing focal points** | Eye confused about where to look | Establish single dominant element |
| **Off-brand colors** | Visual inconsistency, brand dilution | Use exact brand color values |
| **Tiny text** | Unreadable on target platform | Minimum 24px for mobile body text |
| **Cluttered composition** | Information overload | Remove non-essential elements |
| **Poor contrast** | Accessibility issues, strain | Test contrast ratios |

### Accessibility Requirements

- **Color contrast:** Minimum 4.5:1 for body text, 3:1 for large text
- **Text size:** Never below 12px for any readable content
- **Alternative text:** All images require descriptive alt text
- **Color independence:** Information shouldn't rely on color alone
