# Quality Control

Review deliverables against brand standards and brief requirements before final delivery.

## Goal

Every deliverable should reinforce brand identity and meet brief specifications. Catch issues before the client sees them.

## Review Checklist

**Brand Alignment**
- [ ] Colors match brand palette
- [ ] Typography is correct
- [ ] Logo usage follows guidelines
- [ ] Voice/tone matches brand personality

**Brief Compliance**
- [ ] Correct format and dimensions
- [ ] All required elements present
- [ ] Meets objective (does it communicate what it should?)
- [ ] Within scope (not over or under-delivered)

**Technical Quality**
- [ ] Resolution is appropriate
- [ ] File format is correct
- [ ] No visible errors (typos, artifacts, rendering issues)

**Consistency**
- [ ] Multiple assets feel like a cohesive set
- [ ] Cross-platform adaptations maintain core design

## Review Process

1. **Retrieve the brief and brand profile** from memory
2. **Examine each deliverable** against the checklist
3. **Document findings** — what works, what needs adjustment
4. **Provide specific feedback** — not just "fix this" but "the headline should be larger for readability on mobile"

## Feedback Delivery

Be constructive and specific:

- ✓ "The hero image captures the brand energy well. Consider increasing contrast on the headline for better mobile readability."
- ✗ "The headline is hard to read."

## Revision Loop

If issues are found:
1. Route back to the appropriate specialist with specific feedback
2. Track revision count in campaign status
3. Re-review updated deliverables

## Final Approval

When deliverables pass review:
1. Update campaign status to "ready for delivery"
2. Optionally invoke `paw-cra-agent-account-manager` for packaging
3. Log the completed work in memory
