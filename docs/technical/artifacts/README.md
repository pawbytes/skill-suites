# Artifacts Overview

This section documents the output structure and file conventions used by Pawbytes Skill Suites. Understanding these patterns is essential for contributors building skills that integrate with the broader ecosystem.

## What Are Artifacts?

Artifacts are the files and directories generated by skills during execution. They include:

- **Brand workspaces** — Persistent context for each brand
- **Campaign outputs** — Plans, strategies, and execution artifacts
- **Deliverables** — Final outputs like presentations, reports, and media files
- **Configuration** — User preferences and project settings

## Directory Structure

All artifacts live under `.pawbytes/` at the project root:

```
.pawbytes/
├── config/                    # Configuration files
├── marketing-suites/          # Marketing module outputs
├── creative-suites/           # Creative module outputs
└── tools-output/              # Tools module outputs
```

## Path Patterns

Skills use consistent path patterns for output locations:

| Pattern | Example |
|---------|---------|
| `{project-root}/.pawbytes/{module}/` | `.pawbytes/marketing-suites/` |
| `{module}/brands/{brand-slug}/` | `marketing-suites/brands/acme-corp/` |
| `{module}/brands/{brand}/campaigns/{campaign}/` | `marketing-suites/brands/acme-corp/campaigns/launch-q1-2024/` |
| `{module}/reports/{type}/` | `marketing-suites/reports/seo-audits/` |

## Key Concepts

### Brand Workspaces

Each brand gets its own directory containing:

- Brand context and identity
- Marketing plans (SOSTAC)
- Campaign history
- Status tracking

Skills read from brand workspaces to maintain consistency and write to them to preserve progress.

### Configuration Hierarchy

Configuration follows a load order:

1. **Module defaults** — `src/{module}/paw-{module}-setup/assets/module.yaml`
2. **Project config** — `.pawbytes/config/config.yaml`
3. **User overrides** — `.pawbytes/config/config.user.yaml`

Later values override earlier ones. User overrides are gitignored by default.

### Deliverable Naming

Deliverables follow consistent naming conventions:

| Type | Pattern | Example |
|------|---------|---------|
| Brand slug | lowercase-hyphenated | `acme-corp` |
| Campaign slug | `{type}-{descriptor}` | `launch-q1-2024` |
| Phase files | `{number}-{name}.md` | `01-situation.md` |
| Date-stamped | `{name}-{YYYY-MM-DD}.{ext}` | `report-2024-01-15.pdf` |

## Integration Guidelines

When building skills that produce artifacts:

1. **Use resolved paths** — Always resolve `{project-root}` and other variables at runtime
2. **Check before creating** — Read existing files before generating new content
3. **Maintain consistency** — Follow established naming conventions
4. **Update tracking** — Modify status files and READMEs to reflect changes
5. **Respect gitignore** — User-specific configs should not be committed
